# Unsupervised Ontology Reconstruction from Danbooru Meta-Tags Using Robust Association Graphs and SVD Embeddings

Working draft for an academic-style report. Last updated: 2026-05-20.

## Abstract

User-generated tagging systems encode rich semantic structure, but their
long-tailed, noisy, and redundant nature makes this structure difficult to
recover with naive frequency-based methods. This project studies Danbooru 2026
metadata as a large-scale tag transaction corpus. Without using image pixels or
supervised copyright labels during graph construction, we build category-aware
tag vocabularies, compute sparse co-occurrence graphs, score tag associations
with robust statistical metrics, detect communities in the character graph, and
evaluate the resulting topology against held-out copyright metadata.

The pipeline processes 10.62 million posts and constructs a filtered
character-character graph with 28,682 character nodes and 837,806 scored edges.
Association scoring combines lift, PMI, positive PMI, normalized PMI,
directional confidence, and discounted PPMI to reduce popularity bias and rare
co-occurrence artifacts. Louvain community detection recovers franchise-level
and event-level structures: the baseline graph produces 950 exported
communities, with mean held-out copyright purity of 0.953 and median purity of
1.000. A Blue Archive community is recovered with purity 0.984 despite copyright
labels being excluded from clustering.

Finally, the project extends the discrete graph into a representation-learning
baseline by factorizing the discounted PPMI graph with truncated SVD, producing
128-dimensional dense character embeddings. This provides a first bridge from
sparse symbolic graph mining to continuous latent-space retrieval.

## Keywords

Danbooru metadata; association rule mining; PMI; NPMI; sparse matrix
multiplication; community detection; Louvain; tag ontology; representation
learning; SVD embeddings; prompt recommendation.

## 1. Introduction

Danbooru-style metadata is an unusually rich source of semantic supervision:
each post contains user-curated tags that describe characters, copyrights,
artists, visual attributes, poses, objects, and meta properties. Although these
tags are not generated by a formal ontology, the collective tagging behavior of
users and artists induces a latent structure. Characters from the same
franchise co-occur with shared cast members, variants share event and costume
contexts, and visual motifs recur across related entities.

The central question of this project is:

> Can an unsupervised pipeline reconstruct meaningful fandom ontology from
> Danbooru text metadata alone?

The project starts from classical data mining: posts are modeled as
transactions and tags as items. It then builds robust co-occurrence graphs,
scores pairwise associations, and performs graph community detection. The later
stages move toward representation learning by compressing the sparse graph into
dense embeddings. This progression mirrors a larger methodological shift: from
discrete symbolic associations to continuous latent-space representations.

The study deliberately excludes images. It also excludes copyright labels from
the character-character clustering stage, reserving them as held-out domain
knowledge for evaluation. This separation makes it possible to test whether the
topology of character co-occurrence alone can reconstruct intellectual-property
boundaries.

## 2. Data and Preprocessing

### 2.1 Data Source

The experiment uses Danbooru 2026 Parquet metadata from the Hugging Face dataset
`Shio-Koube/Danbooru-2026-parquet-metadata`. Only text metadata is used. Raw
image files are not downloaded or processed.

The local raw input is expected under:

```text
data/raw/danbooru-2026/
```

Generated artifacts remain under ignored `data/processed/` directories and are
not committed to the repository.

### 2.2 Category-Aware Tag Modeling

Danbooru tags are category-specific. A tag string can represent a general visual
attribute, a character, a copyright, an artist, or metadata. The preprocessing
pipeline preserves this distinction by treating the key as `(category, tag)`,
rather than as a raw string alone. This prevents cross-category homonym
contamination.

For the main character graph, the preprocessing stage selects active posts,
filters out deleted and banned posts, splits whitespace-separated tag strings,
removes null and empty tokens, and applies a minimum tag frequency threshold.
The default character vocabulary threshold is:

```text
min_tag_count = 50
```

After filtering, the character vocabulary contains 28,682 tags.

### 2.3 Sparse Relational Representation

The processed data is represented by three core tables:

```text
posts.parquet       post_idx -> post_id
tag_vocab.parquet   tag_id, category, tag, count
post_tags.parquet   post_idx, tag_id, category
```

This structure avoids materializing a quadratic post-level self-join. Instead,
category-specific sparse post-tag matrices are constructed and multiplied with
SciPy:

```text
X_left.T @ X_right
```

For `character-character`, this yields a sparse co-occurrence graph without
building a large intermediate Cartesian table.

## 3. Methodology

### 3.1 Association Metrics

Let `N` be the number of valid posts, `count_a` and `count_b` be the marginal
post counts of tags `a` and `b`, and `co_count` be the number of posts where
both tags occur. The pipeline computes:

```text
P(a)  = count_a / N
P(b)  = count_b / N
P(a,b)= co_count / N
```

Lift is defined as:

```text
lift(a,b) = P(a,b) / (P(a)P(b))
```

PMI and PPMI are:

```text
pmi(a,b)  = log2(lift(a,b))
ppmi(a,b) = max(pmi(a,b), 0)
```

Normalized PMI bounds the score into a more comparable range:

```text
npmi(a,b) = pmi(a,b) / -log2(P(a,b))
```

The degenerate full-dataset binding case `P(a,b)=1` is assigned `npmi=1.0`.

The pipeline also computes directional confidence:

```text
confidence(a -> b) = co_count / count_a
confidence(b -> a) = co_count / count_b
```

To reduce rare-pair inflation, discounted PPMI is used for ranking:

```text
discounted_ppmi = ppmi * co_count / (co_count + k)
```

with default `k=10`.

### 3.2 Character Co-Occurrence Graph

The main graph is an undirected weighted graph over character tags. Edges are
retained after minimum support filtering:

```text
min_pair_count = 5
```

Same-category edges store only one row with `tag_a_id < tag_b_id`, while graph
algorithms treat the edge as undirected. The full scored character-character
edge table contains 837,806 rows.

### 3.3 Community Detection

Community detection uses NetworkX's Louvain implementation with
`discounted_ppmi` as edge weight. The baseline setting is:

```text
min_npmi = 0.15
min_co_count = 15
resolution = 1.2
seed = 42
```

Two stricter configurations are used to test threshold sensitivity:

| Run | min NPMI | min co-count | resolution |
| --- | ---: | ---: | ---: |
| baseline | 0.15 | 15 | 1.2 |
| strict | 0.50 | 25 | 1.2 |
| fine | 0.60 | 25 | 1.5 |

### 3.4 Held-Out Copyright Evaluation

Copyright tags are not used during character-character graph construction or
community detection. They are used afterward as held-out domain labels.

For each character, the dominant copyright is the copyright tag with the largest
character-copyright co-occurrence count across posts. For each detected
community, purity is defined as:

```text
purity(C_i) = max_d |C_i intersect d| / known_members(C_i)
coverage(C_i) = known_members(C_i) / |C_i|
```

This evaluates whether an unsupervised character topology reconstructs
franchise boundaries.

### 3.5 Prompt-Oriented Query Layer

The graph artifacts are also used for an offline prompt recommendation
prototype. Two interaction modes are implemented:

1. Neighbor recommendation over the character graph.
2. Post-level conditional character queries from general tags.

The second mode supports queries such as:

```powershell
danbooru-graph query-characters `
  --include-general "dark-skinned_male,dark-skinned_female" `
  --female-only `
  --top-k 50
```

The `--female-only` filter uses an empirical profile inferred from `solo +
1girl` and `solo + 1boy` signals. This is a metadata-derived heuristic rather
than a canonical gender database.

### 3.6 SVD Representation Learning Baseline

The Phase 6 representation-learning baseline factorizes the symmetric
discounted PPMI character graph with truncated SVD. Given a sparse matrix `M`
where:

```text
M[i,j] = discounted_ppmi(tag_i, tag_j)
M[j,i] = M[i,j]
M[i,i] = 0
```

the model computes:

```text
M ~= U S V^T
embedding_i = U_i * sqrt(S)
```

Rows are L2-normalized after factorization, allowing cosine similarity to be
computed as a dot product. The default dimension is:

```text
dim = 128
```

The current full run produces:

```text
embeddings.npy shape: (28682, 128)
solver: PROPACK
all vectors finite: true
row norms: approximately 1.0
```

This baseline follows the connection between skip-gram embeddings and implicit
matrix factorization, while avoiding neural training dependencies in the first
representation-learning stage.

## 4. Experiments and Results

### 4.1 Pairwise Semantic Associations

The highest-scoring character-character edges reveal hard semantic bindings:

1. Inseparable duos and twins, such as paired mascots or sibling characters.
2. Character variants and form transformations.
3. Entity-companion or entity-summon relationships.

These edges typically have high NPMI and near-symmetric confidence, indicating
that the tags are almost always used together.

### 4.2 Asuna/Karin Case Study

The `asuna_(blue_archive)` and `karin_(blue_archive)` pair illustrates why raw
co-occurrence counts are insufficient. Their observed co-occurrence is much
higher than random expectation, giving high lift. However, Asuna's large
marginal frequency reduces both directional confidence and NPMI relative to
near-inseparable duos. This is a useful example of hub-node penalty: globally
popular characters can have many meaningful but non-exclusive relationships.

### 4.3 Community Detection Results

| Run | Exported communities | Mean purity | Median purity | Purity >= 0.9 |
| --- | ---: | ---: | ---: | ---: |
| baseline | 950 | 0.953 | 1.000 | 87.8% |
| strict | 1,237 | 0.967 | 1.000 | 90.7% |
| fine | 1,745 | 0.968 | 1.000 | 90.2% |

The baseline graph recovers a Blue Archive community:

```text
community_id: 12
size: 377
dominant copyright: blue_archive
dominant copyright members: 371
purity: 0.984
```

This supports the main hypothesis: character-character co-occurrence topology
alone can reconstruct high-level franchise boundaries.

### 4.4 Micro-Community Structure

When thresholds become stricter, broad IP-level clusters decompose into smaller
event and variant clusters. A C&C-like Blue Archive subcluster contains
Akane/Karin/Neru/Toki/Asuna variants and is summarized by general tags such as:

```text
cleaning_&_clearing_(blue_archive)
sukajan
aqua_leotard
sig_mpx
```

This suggests that threshold sweeps reveal different semantic resolutions:
relaxed graphs recover franchises, while stricter graphs recover motifs,
costume events, and tightly co-drawn character variants.

### 4.5 Conditional Character Query Results

The post-level query layer can answer multi-tag questions that the
character-character graph alone cannot. For the query:

```text
dark-skinned_male AND dark-skinned_female
```

with empirical female filtering enabled, high-count candidates include:

| Character | Co-count |
| --- | ---: |
| `nessa_(pokemon)` | 179 |
| `nitocris_(fate)` | 148 |
| `bea_(pokemon)` | 131 |
| `karin_(blue_archive)` | 125 |
| `chloe_von_einzbern` | 93 |
| `mallow_(pokemon)` | 88 |
| `karin_(bunny)_(blue_archive)` | 71 |
| `dehya_(genshin_impact)` | 67 |

This extends the project from graph mining into practical tag retrieval.

### 4.6 SVD Embedding Results

The SVD embedding run compresses the character graph into 128-dimensional dense
vectors. For `asuna_(blue_archive)`, nearest neighbors are dominated by Blue
Archive tags and variants. For `karin_(blue_archive)`, nearest neighbors also
fall within the Blue Archive region, including C&C-adjacent and variant tags.

The similarity between Asuna and Karin in the SVD space is:

```text
cosine(asuna_(blue_archive), karin_(blue_archive)) = 0.999926
```

This confirms that the embedding captures broad franchise-level topology.
However, the very high cosine similarities across many Blue Archive characters
also indicate that the first SVD baseline is dominated by large IP-level
structure. Future work should test alternative weighting, removal of dominant
components, multi-category graphs, and Item2Vec/Node2Vec baselines to improve
micro-motif sensitivity.

## 5. Discussion

The results suggest that UGC tag systems encode multiple semantic layers:

1. Pairwise hard bindings recover inseparable duos, variants, and entity-form
   relationships.
2. Medium-threshold graph communities reconstruct franchise-level ontology.
3. Strict graph thresholds expose event-driven and costume-driven motifs.
4. Dense SVD embeddings capture high-order similarity beyond direct edge lookup,
   but initially emphasize large global clusters.

This layered behavior is useful for both research and application. In a prompt
recommendation system, different scoring modes serve different user needs:
confidence can autocomplete canonical traits, discounted PPMI can suggest
high-information associations, community summaries can inject motifs, and dense
embeddings can support nearest-neighbor discovery.

## 6. Limitations

This draft reports a character-only main graph. As a result, analogy-style
queries such as:

```text
asuna - blue_archive + arknights
```

cannot yet be evaluated, because copyright and general tags are not present in
the SVD embedding vocabulary. A multi-category heterogeneous graph is required
for that experiment.

The held-out copyright evaluation is strong for franchise reconstruction but
does not fully measure subgroup quality, costume semantics, or visual style.
The gender profile used in conditional queries is empirical and may fail for
ambiguous, gender-swapped, ensemble, or non-human characters.

The SVD baseline is deterministic and efficient, but its first components may
overrepresent large global franchises. Future experiments should include
component removal, shifted PPMI variants, Item2Vec, Node2Vec, and approximate
nearest-neighbor indexing.

## 7. Ethical and Dataset Considerations

The project uses text metadata only and does not redistribute raw Danbooru
images or generated large artifacts. Generated tables, graphs, and embeddings
remain under ignored local data directories. The analysis should be interpreted
as a study of user-generated metadata behavior, not as a canonical ontology of
the underlying fictional works.

Because Danbooru contains adult and sensitive content categories, downstream
applications should include appropriate filtering, user controls, and dataset
documentation. Tag associations may reflect community biases, popularity
effects, and uneven coverage across franchises.

## 8. Reproducibility

Install the project:

```powershell
python -m pip install -e ".[dev]"
python -m pytest -q
```

Build the character graph:

```powershell
danbooru-graph prepare-vocab --input data/raw/danbooru-2026 --out data/processed --categories character
danbooru-graph build-edges --processed data/processed --pair character-character --min-pair-count 5
danbooru-graph score-edges --processed data/processed --pair character-character --discount-k 10 --sort-by discounted_ppmi --top-k 50
```

Detect communities:

```powershell
danbooru-graph detect-communities --processed data/processed --min-npmi 0.15 --min-co-count 15 --resolution 1.2
```

Evaluate held-out copyright purity:

```powershell
danbooru-graph build-copyright-profile --input data/raw/danbooru-2026 --out data/processed/evaluation
danbooru-graph evaluate-purity --communities data/processed/communities/character_communities_npmi0.15_co15_res1.2.json --profile data/processed/evaluation/character_copyright_profile.parquet
```

Build SVD embeddings:

```powershell
danbooru-graph build-embeddings --processed data/processed --pair character-character --method svd --dim 128
danbooru-graph nearest-tags --embeddings data/processed/embeddings/character_character_svd_d128 --tag "asuna_(blue_archive)" --top-k 20
```

## 9. Future Work

1. Build a heterogeneous graph over character, copyright, and general tags.
2. Evaluate analogy tasks in a multi-category embedding space.
3. Train Item2Vec directly from post-level tag sets.
4. Train Node2Vec over the weighted character graph and compare it against SVD.
5. Add FAISS for scalable nearest-neighbor retrieval once vector count grows.
6. Use UMAP or t-SNE to visualize the embedding manifold.
7. Define quantitative retrieval benchmarks using held-out copyright and
   faction labels.
